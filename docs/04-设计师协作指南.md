# 🎨 设计师协作指南

> 适用对象：视觉设计师、交互设计师  
> **你不需要写任何代码。** 你的专长是视觉和体验，这份指南帮你用最熟悉的方式，把你的设计接入工作台。

---

## 一、核心分工

| 你负责 | 朱彦霖负责 |
|--------|-----------|
| 视觉创作：配色、布局、图片、文案 | 技术实现：把你的设计转换成可运行的代码 |

## 二、两种协作路径（选一种）

### 路径 A：Figma 组件库 + 插件导出（推荐）

**适合：** 习惯用 Figma 的设计师

**流程：**

```
你在 Figma 里设计 → 使用朱彦霖提供的组件库 → 插件一键导出 → 发给朱彦霖
```

**具体步骤：**

1. **启用团队组件库**  
   朱彦霖会提供一个 Figma 团队库链接，启用后你能看到所有已有组件（Banner、老虎机、楼层条等）

2. **用组件库拼装页面**  
   在你的 Figma 文件里，用这些组件搭建页面。可以自由调整文案、配色、图片，但请使用库里的组件，不要用矩形自己画新 UI

3. **插件导出**  
   安装「闪购设计导出」插件（朱彦霖提供），点击「导出模板」，自动生成 `.template.json` 文件

4. **发给朱彦霖**  
   把 `.json` 文件通过钉钉/微信发给朱彦霖，他会集成进工作台并通知你

**为什么这样做？**  
你的 Figma 文件里记录了"用了什么组件 + 配置是什么"，插件把这份信息转换成工作台能读懂的格式。你看到的还是 Figma，但产出已经是可直接上线的配置。

---

### 路径 B：配置表单（最简单）

**适合：** 不用 Figma、或只是想快速配置一个标准页面

**流程：**

```
填一个在线表单 → 系统自动生成模板 → 朱彦霖导入工作台
```

**步骤：**

1. 打开配置表单（链接由朱彦霖提供）
2. 按照表单填写：背景图、主标题、副标题、按钮文案、配色方案
3. 点击「生成模板」，自动下载 `.json` 文件
4. 发给朱彦霖

**表单示例（会长这样）：**

| 配置项 | 你填写的内容 |
|--------|------------|
| 页面名称 | 618 主会场 |
| 头图背景 | [上传图片] |
| 主标题 | 狂欢 618 |
| 配色方案 | 大促红 / 活力橙 / 品牌黄 |
| 顶部组件 | 老虎机抽奖 |
| 是否需要倒计时 | 是，结束时间：6月20日 |

## 三、你的设计边界

**可以自由调整的：**
- ✅ 所有文案内容
- ✅ 图片和背景图
- ✅ 在提供的配色方案内选择颜色
- ✅ 组件的排列顺序
- ✅ 是否显示某个组件

**需要跟朱彦霖确认的：**
- ⚠️ 全新的 UI 元素（组件库里没有的）
- ⚠️ 超出规范的自定义配色（保证品牌统一性）
- ⚠️ 复杂的交互逻辑

**不需要你操心的：**
- ❌ 代码实现
- ❌ 切图尺寸和命名
- ❌ 多端适配
- ❌ JSON 格式

## 五、常见问题

**Q：我用的是 MasterGo / 蓝湖，不是 Figma，怎么办？**  
A：走路径 B（配置表单），或者截图 + 标注发给朱彦霖，他来转换。

**Q：我的设计有个组件库里没有的效果，怎么办？**  
A：通过 [需求漏斗](./02-需求漏斗.md) 提交，附上效果参考。如果有活动背景且值得复用，朱彦霖会开发后加入组件库。

**Q：我提交的设计最终长什么样？**  
A：朱彦霖集成后会在工作台里发给你预览链接确认，确认通过后才正式加入模板库。

**Q：我的模板被其他人用了，我能知道吗？**  
A：能。每次你贡献的模板被复用，你会收到通知。使用次数越多，说明你的设计越有价值。
