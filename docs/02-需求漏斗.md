# ⚠️ 需求提交入口 · 必读

> **提交前请确认三件事：**
> 1. 你**正在或即将**使用本工具完成一个真实活动（不是"以后可能用"）
> 2. 你已看过 [使用指南](./01-使用指南.md)，确认现有功能**确实无法满足**
> 3. 你能用**一句话**说清楚你要解决的问题

不满足以上三点的需求，将不被处理。

---

## 📋 提交模板（复制填写后发到飞书表格）

**需求标题**：【一句话概括，如"希望楼层条能支持自定义字体大小"】

**用户故事**：
```
作为【你的角色，如"运营"】，
当我想【做某件事，如"区分大促主场和子场"】时，
我希望【达到什么效果，如"主场楼层条字体更大"】，
以便【实现什么价值，如"视觉层级更清晰"】。
```

**界面草图（必填）**：
- 用任何方式给出草图：截图+箭头标注、纸上画拍照、绘图软件均可
- 不需要美观，能看懂即可
- **没有草图的需求，不被处理**

**活动背景**：
- 这个需求是为了哪个活动？（如"618 大促主场，6月15日上线"）
- 时间敏感度？（一般 / 重要 / 紧急-影响上线）

**成功标准**：
- 功能上线后，你怎么判断它满足了你的需求？
- 示例："我能在 30 秒内自己调出想要的字体大小，不需要再找设计师"

## 📬 提交方式

1. 将上述内容填入飞书需求表格（链接由朱彦霖发给你）

3. **紧急需求**（影响当前活动上线）→ 直接联系朱彦霖，并附活动页面链接

---

## 📏 优先级规则

我会优先处理满足以下条件的需求（按权重排序）：

| 优先级 | 条件 |
|--------|------|
| 🔴 最高 | 影响活动正常上线 |
| 🟠 高 | 多个业务方同时提出 |
| 🟡 中 | 有明确用户故事 + 草图 |
| 🟢 低 | 模糊建议 / 锦上添花 |

> 需求可能被拒绝、修改或延期，我会在表格中说明原因。  
> **"我觉得可以加个 XX 功能"这类无背景的建议，不被处理。**

## ❌ 典型被拒示例

| 提交内容 | 拒绝原因 |
|---------|---------|
| "能不能加个动画效果" | 没有用户故事、没有草图、没有活动背景 |
| "界面能不能更好看" | 过于模糊，无法转化为可执行的需求 |
| "参考竞品 X 做一样的" | 没有说明具体场景和价值 |
| "以后可能用到，先做着" | 不基于真实活动，不处理 |
