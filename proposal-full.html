优化规划（Part 02-07）

基于深海圈 AI 工具箱方法论的系统优化

02
描述需求 · SPEC 规范化
把模糊想法写成清晰、可执行的 SPEC
P0 🔥
  • 给每个组件写 SPEC 五要素
    问题陈述 / 方案描述 / 技术约束 / Non-goals / 成功标准
  • 画功能逻辑地图
    按 CAP / PAGE / UI / ACT / FLOW / API / STORE / RULE / TEST 9 类节点拆解
  • 沉淀 CLAUDE.md 项目记忆
    让 AI 知道双仓库架构、组件规范、动效规则
03
代码开发 · 积木思维沉淀
用积木思维快速搭建，沉淀 SKILL.md
P0 🔥
  • 沉淀 SKILL.md 组件规范
    已创建 skills/shangou-component.md，描述 7 个组件的入参、样式规范、动效配置
  • MasterGo → 代码生成模板优化
    按 Section/Component 拆分复刻，让生成的代码结构更规范
04
Bug 调试 · 测试兜底
测试先行 + 把检查交给 hook/subagent 自动跑
P1
  • 给组件生成逻辑加测试
    验证生成的 JSON 结构是否符合规范、Canvas 输出是否正确
  • 配置 Stop hook
    每次 AI 改完代码自动跑测试，不过不许收工
  • 用 task-evaluator subagent 评审代码
    创建只读评审专家，检查 PLAN/CODE/TEST 三类产物
05
上线基础设施
离开扣子编程沙箱，进入深水区
P2
  • 保存用户设计历史
    需要数据库（PostgreSQL / Supabase）
  • 导出图片托管
    需要 Cloudflare R2 存储（AI 生图临时链接转永久）
  • 用户登录系统
    better-auth / next-auth，支持保存个人配置
07
自动化规模化 · AI 编队
把 AI 变成员工和编队，规模化干活
P1
  • 浏览器自动化验证
    用 Chrome DevTools MCP 自动打开 Figma，验证导出效果
  • AI 团队架构
    ① 组件代码生成 agent — 只负责 DSL 转代码
    ② 视觉还原评审 agent — 只读，对比设计稿和生成结果
    ③ 动效逻辑检查 agent — 验证动画参数是否符合规范
  • 批量抓取竞品组件库
    用 Browser Use 批量调研竞品，生成参考报告
